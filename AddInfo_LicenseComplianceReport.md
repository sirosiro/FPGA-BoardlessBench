# FPGA-BoardlessBench (F-BB) ライセンス・コンプライアンス報告書

## 1. 概要
FPGA-BoardlessBench (F-BB) は **MIT ライセンス** で公開されています。本プロジェクトが依存する外部ツールやライブラリとのライセンス整合性、およびユーザーが利用・配布する際の注意点について整理しました。

## 2. 依存ツールとの互換性評価
F-BB が「プリリクエスト」として要求する主要ツールとのライセンス親和性は以下の通りです。

| カテゴリ | ツール | ライセンス | 評価 | 根拠・注意点 |
| :--- | :--- | :--- | :--- | :--- |
| **開発環境** | GCC / G++ / Make | GPL v3 | **OK** | 独立したツールとしての利用。生成バイナリへの波及なし。 |
| **ランタイム** | Python / Node.js | PSF / MIT | **OK** | MIT ライセンスと完全な互換性あり。 |
| **シミュレータ** | Verilator | LGPL v3 / Artistic 2.0 | **OK*** | F-BB は Verilator を外部プロセスとして呼び出すのみ。 |
| **可視化** | GTKWave | GPL v2+ | **OK** | 外部ビューアとしての利用。 |

> [!IMPORTANT]
> **Verilator に関する特記事項**:
> F-BB が生成する C++ コードは Verilator のヘッダーをインクルードします。ユーザーが生成されたシミュレーションバイナリを再配布する場合、Verilator のライセンス（LGPL v3 または Artistic 2.0）が適用される可能性がある点に留意してください。

## 3. 実装上のライセンス構成

### A. 外部ライブラリ (npm)
ダッシュボード（Node.js / React）で使用している主要ライブラリ（`express`, `socket.io`, `react` 等）は、すべて MIT または互換ライセンス（ISC, BSD 等）であり、MIT ライセンスでの配布に支障はありません。

### B. Shim 層 (LD_PRELOAD) の扱い
F-BB の Shim ライブラリ (`libfpgashim.so`) は、ターゲットアプリケーションと実行時に動的リンクされます。MIT ライセンスを適用することで、ユーザーのアプリケーションがどのようなライセンス（商用・GPL 等）であっても、その利用を制限することはありません。

### 第三者 IP の保護
F-BB はユーザーが提供する Verilog ソースファイルを読み込みます。
*   **商用 IP の取り扱い**: FPGA ベンダーが提供する有償 IP や、機密保持契約（NDA）下にある設計データをシミュレーションに使用する場合、それらのファイルを F-BB と共に公開リポジトリ等へアップロードしないよう十分注意してください。
*   **免責**: 本プロジェクトの利用によって生じた、第三者の知的財産権侵害に関するいかなる紛争についても、開発者は一切の責任を負いません。

F-BB を MIT ライセンスで公開することに法的な障壁はなく、オープンな開発エコシステムに適した構成となっています。
