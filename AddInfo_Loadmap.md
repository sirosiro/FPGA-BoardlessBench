# FPGA-BoardlessBench (F-BB)ロードマップ

Zynq-7000 SoCのプロセッサ・サブシステム（PS）に内蔵されている主要なペリフェラルを網羅し、`FPGA-BoardlessBench (F-BB)`の現在のアーキテクチャ（骨格）との親和性・対応状況を整理しました。

---

## Zynq PS ペリフェラル対応状況

ZynqのPSには、ハードウェアIPとして固定されている多くのペリフェラルがあります。これらを「FPGA-BoardlessBench (F-BB)での再現難易度」と「開発における重要度」で分類します。

| ペリフェラル | Zynqでの本数/仕様 | FPGA-BoardlessBench (F-BB) 対応状況 | エミュレーションの方向性 | 優先度 |
| :--- | :--- | :--- | :--- | :--- |
| **GPIO (MIO/EMIO)** | 最大118本 | **対応済** | 118ch双方向、LED/トグルスイッチUI、DATA2対応 | 特高 |
| **UART** | 2ポート | **対応済** | PTYリダイレクト、マルチポート同時監視、TCPブリッジ | 特高 |
| **I2C** | 2ポート | **対応済** | Shim Interceptor によるバス識別と Stub 応答 | 高 |
| **QSPI Flash** | 1ポート | 未対応 | `mmap` による仮想イメージの読み出し | 中 |
| **Ethernet (GEM)** | 2ポート | 未対応 | TAP/TUNデバイスによるホスト通信 | 低 |
| **USB 2.0 (OTG)** | 2ポート | 未対応 | 仮想USBスタックへのブリッジ | 低 |
| **SD/SDIO** | 2ポート | 未対応 | ローカルディレクトリのマウント | 中 |
| **SPI** | 2ポート | 未対応 | Pythonによる仮想ADC/DAC Stub | 高 |
| **CAN** | 2ポート | 未対応 | SocketCANへのマッピング | 低 |
| **WDT (Watchdog)** | 3個 | 未対応 | タイムアウトイベントの通知 | 低 |
| **TTC (Triple Timer)** | 2個 | 未対応 | クロック精度のタイマーイベント | 中 |

## FPGA-BoardlessBench (F-BB) 次の道標

シミュレーション環境が「骨格」として完成している今、次に目指すべきは **「FWエンジニアが実機を触っていると錯覚するほどの利便性」** の獲得です。

### 達成済みのマイルストーン (Completed Milestones)
*   **ペイン配置の自由化とリサイズ高度化 (Dockview の導入):** 各パネル（Register, GPIO, Terminal, Tracer）をドラッグ＆ドロップで自由に配置換え、フローティング、および分割レイアウトできる完全な自由配置 UI を実現。
*   **ヒストリカル・トレースの統合 (Register State Tracer):** ブラウザ上でレジスタ値やGPIO状態の論理的推移をリアルタイムに視覚化・監視するトレーサー機能。
*   **マルチUARTの統合・タブ表示:** 複数の仮想 UART コンソールのタブ切り替えおよび同時監視に対応。
*   **動的構成変更:** DTS の変更に伴う `board_manifest.json` のポーリング自動ロードとUIへの即時反映。

### 1. 短期目標：デバッグ視認性の極大化（Dashboard 2.0）
*   **UI側でのデバイス個別リセット機能:** 
    ダッシュボード上のUIボタンから、各仮想デバイス（GPIOやUART等）のステートや割り込み状態を、個別に初期状態へ強制リセットする機能を追加する。


### 2. 中期目標：外部インターフェースの「仮想スタブ化」
実機デバッグで最も時間がかかる「外付けデバイスとの通信」を仮想化します。
*   **I2C/SPI Bridge:** 
    Verilog側のI2C/SPI信号をPythonに吸い上げ、Python側で温度センサやEEPROMの挙動をシミュレートする。これにより、デバイスが届く前にドライバを完遂できます。
*   **Interrupt (IRQ) の可視化:** 
    PLからPSへの割り込み信号（IRQ_F2P）が飛んだ瞬間をダッシュボード上で点灯させ、FW側のハンドラが何ミリ秒後に反応したかを計測する。

### 3. 長期目標：エコシステムの構築（Device Tree 連携）
先ほど議論した「.v からの定義抽出」を具体化します。
*   **Auto-DTSI Generator:** 
    Verilogのパラメータから、Linux用のデバイスツリー断片（.dtsi）を自動生成。
*   **UIO Configurator:** 
    生成されたアドレスマップに基づき、シミュレーション環境のメモリマップを自動構成。

## FPGA-BoardlessBench (F-BB)が提供すべき価値

Zynq開発における「魔の期間（基板待ち、Vivado合成待ち）」を埋めるための最強の武器は、**「Linuxアプリが、FPGAの中身を意識せずに叩ける環境」** です。

次に注力すべきは、**「マルチUART対応」**と**「GPIO入力スイッチ対応」**を皮切りとした、**「ユーザーが独自の仮想ペリフェラル（I2Cデバイス等）を簡単に追加できる口（プラグイン構造）」** の整備でしょう。

これが実現すれば、FPGA-BoardlessBench (F-BB)は「シミュレータ」ではなく、Zynq開発における「仮想マザーボード」へと昇華します。

> [!CAUTION] ここから先は大風呂敷を広げてます。話半分で読んでください。以下の内容を実現するには現時点のFPGA-BoardlessBench (F-BB)が実現しているペリフェラルの種類が全く足りていません。

さらに、FPGA-BoardlessBench (F-BB)は単なる学習用シミュレータに留まらず、実機評価ボードの **「ハードウェアの物理的制約からの解放」** という極めて実戦的な価値を提供します。

*   **ハイエンド環境の民主化**

    ZC706（約45万円）のようなフラグシップ評価ボードでなければ試せなかった「全ペリフェラル同時使用」や「大規模ロジック検証」を、ノートPC一台のローカル環境で実現します。予算や基板の有無という「エンジニアの外部要因」を排除し、純粋な「設計能力」をフルに発揮できる場を提供し、 **「本来なら巨大な Vivado 環境と高価なボードが必要な Zynq 開発の 9 割（ロジックの機能検証とドライバ開発）を、電車の中のラップトップで終わらせる」** ことを可能にします。

*   **「論理的全部入り」による先行開発**

    安価な実機ボードではピンアサインの競合で諦めていた構成も、FPGA-BoardlessBench (F-BB)上では「論理的に正しい理想の状態」で構築可能です。基板設計が固まるのを待つのではなく、理想のインターフェース仕様に基づいた「ロジックの蒸留」を先行させることで、実機投入時の手戻りをゼロに近づけます。

*   **可視化による「実機超え」のデバッグ**

    物理基板ではオシロスコープやILA（論理アナライザ）を駆使しても見えない「チップ内部の全サイクルの挙動」を、透過的に観測可能です。これは「実機の再現」ではなく、**「実機よりも深い洞察を得られるデジタルツイン」** としての価値を意味します。

*   **AMP（非対称マルチプロセッシング）環境の仮想構築と結合テスト（ベアメタル対応済）**

    近年主流となっている「コア0でLinux、コア1でRTOS/ベアメタル」を協調動作させるAMPアーキテクチャ（OpenAMPなど）は、実機環境でもデバッグが極めて困難な領域です。FPGA-BoardlessBench (F-BB)では「FW＝単なるPC上のプロセス」として扱えるという大きな強みを活かし、Linux用プロセスとベアメタル/RTOS用プロセスを複数同時に立ち上げ、ホストPCのマルチコア処理を利用して「1つの仮想FPGA」へ同時にアクセスさせるシミュレーションが可能です。

    現在、仮想 Remoteproc デーモンによる起動フックと、C Shim の `MAP_FIXED` 自動インジェクションにより、Mコアのコード透過性100%を維持したまま、実機同様の物理アドレス空間を直接操作する**ベアメタルファームウェアとの協調シミュレーション**、および動的ホットスワップ検証環境（シナリオ09）が実装されています（FreeRTOS などの RTOS との協調動作は今後のマイルストーンとなります）。（詳細: **[BBにおけるRTOS/ベアメタルFWエミュレーション](AddInfo_RTOS.md)**）


FPGA-BoardlessBench (F-BB)は将来的には「安価な代替品」ではなく、 **「実機（物理）という不自由な環境を触る前に、エンジニアが思考の限りを尽くして最高品質のロジックを練り上げるための、仮想フラグシップ・プラットフォーム」** としての価値を提供することを目指します。
